# CityVoice: как запустить проект

Эта инструкция нужна, если вам скинули проект архивом `.rar` или обычной папкой.

## Что нужно скачать заранее

Скачайте и установите:

1. `Python 3.13.x (64-bit)`
2. `PostgreSQL 16+`
3. Любой архиватор, если проект пришёл в `.rar`
   Подойдут `WinRAR` или `7-Zip`

Рекомендуемая версия Python для этого проекта:

- `Python 3.13`

При установке Python обязательно поставьте галочку:

- `Add python.exe to PATH`

## Что НЕ нужно скачивать отдельно

Не нужно отдельно ставить:

- `Flask`
- `Flask-Mail`
- `python-dotenv`
- `.venv`

Проект сам установит нужные зависимости при запуске.

В архиве для друга обычно НЕ нужны:

- `.venv`
- локальная база данных
- временные `.log` файлы

`.env` может быть в архиве, а может и не быть:

- если `.env` уже есть, почта для сброса пароля должна работать сразу;
- если `.env` нет, можно взять `.env.example` и переименовать его в `.env`.
- если в архиве уже есть папка `instance` с `cityvoice.db`, проект поднимется с готовым админом и демо-данными.

## Как запустить проект

1. Распакуйте архив в любую папку.
2. Откройте папку `CityVoice`.
3. Убедитесь, что ваш `PostgreSQL` уже запущен.
4. В адресной строке проводника введите `powershell` и нажмите `Enter`.
5. В открывшемся окне PowerShell выполните:

```powershell
.\start.ps1
```

Если в архиве уже есть `.env`, этот шаг можно пропустить.

Если `.env` нет и нужен сброс пароля по почте, перед первым запуском:

1. скопируйте файл `.env.example` в `.env`
2. заполните в `.env` почтовые настройки

Минимальный пример:

```env
MAIL_USERNAME=your_gmail@gmail.com
MAIL_PASSWORD=your_16_char_app_password
CITYVOICE_MAIL_SENDER=CityVoice <your_gmail@gmail.com>
MAIL_SERVER=smtp.gmail.com
MAIL_PORT=587
MAIL_USE_TLS=true
MAIL_USE_SSL=false
```

Если `.env` не создавать, сайт всё равно запустится, но письма для сброса пароля отправляться не будут.

После этого скрипт автоматически:

- создаст `.venv`, если его нет;
- обновит `pip`;
- установит зависимости из `requirements.txt`;
- запустит сайт.

Когда увидите, что сервер запущен, откройте в браузере:

```text
http://127.0.0.1:5000
```

## Если PowerShell не даёт запустить скрипт

Иногда Windows блокирует запуск `.ps1`.

Тогда вместо обычной команды выполните:

```powershell
powershell -ExecutionPolicy Bypass -File .\start.ps1
```

Или просто запустите:

```cmd
start.cmd
```

Если в проекте лежит чужой `.venv` из другого компьютера, `start.ps1` теперь сам пересоздаст его заново.

## Если хочется запускать вручную

Можно и так:

```powershell
py -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe app.py
```

Но обычно это не нужно, потому что `.\start.ps1` делает всё сам.

## Тестовый вход

Если нужен готовый тестовый пользователь:

- email: `aidana@example.com`
- пароль: `user123`

Администратор:

- email: `admin@cityvoice.local`
- пароль: `admin123`

Дополнительно в демо-базе могут уже быть:

- тестовый модератор `moderator@cityvoice.local` с паролем `moderator123`
- несколько обычных пользователей с паролем `user123`
- набор жалоб и петиций для демонстрации интерфейса

## Важно

- Пока окно PowerShell открыто, сайт работает.
- Если закрыть PowerShell, сервер остановится.
- База данных хранится локально в `%LOCALAPPDATA%\CityVoice\instance\cityvoice.db`.
- Если `%LOCALAPPDATA%` недоступен, используется запасной путь `instance/cityvoice.db` рядом с проектом.

## Самый короткий вариант

Если совсем кратко:

1. Установить `Python 3.13`
2. Распаковать проект
3. Открыть PowerShell в папке проекта
4. Выполнить:

```powershell
.\start.ps1
```

5. Открыть:

```text
http://127.0.0.1:5000
```

## Как открыть сайт другу через Cloudflare

Если нужно показать проект другу через интернет, можно поднять временную публичную ссылку через `Cloudflare Tunnel`.

### Что нужно

Нужны 2 открытых окна PowerShell:

1. первое окно — сам сайт;
2. второе окно — `cloudflared`.

Пока оба окна открыты, ссылка для друга работает.

### Шаг 1. Запустите сам проект

В первой консоли PowerShell, внутри папки `CityVoice`, выполните:

```powershell
.\start.ps1
```

После запуска сайт должен открываться локально:

```text
http://127.0.0.1:5000
```

### Шаг 2. Запустите Cloudflare Tunnel

Если в папке проекта уже есть `cloudflared.exe`, ничего скачивать не нужно.

В лёгкой отправляемой копии этот файл может отсутствовать специально, чтобы архив был меньше.

Если файла нет, скачайте его по ссылке:

```text
https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-windows-amd64.exe
```

Сохраните файл в папку проекта и назовите его:

```text
cloudflared.exe
```

Во втором окне PowerShell, внутри папки `CityVoice`, выполните:

```powershell
.\cloudflared.exe tunnel --url http://127.0.0.1:5000
```

После этого в консоли появится ссылка вида:

```text
https://xxxxx.trycloudflare.com
```

Именно её можно отправлять другу.

### Важно про Cloudflare

- ссылка временная;
- при новом запуске адрес обычно меняется;
- если закрыть окно с `cloudflared`, ссылка перестанет работать;
- если закрыть окно с сервером, сайт тоже перестанет открываться;
- иногда после запуска нужно подождать 5-15 секунд, пока ссылка станет доступной.
- ссылки для сброса пароля через `trycloudflare` теперь обычно собираются автоматически;
- вручную задавать `CITYVOICE_BASE_URL` обычно не нужно;
- `CITYVOICE_BASE_URL` нужен только если вы сами открыли сайт по `http://127.0.0.1:5000`, но хотите, чтобы письмо вело на внешний адрес Cloudflare.
