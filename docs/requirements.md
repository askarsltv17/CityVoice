# Требования к проекту CityVoice

## 1. Назначение системы

`CityVoice` предназначен для публикации городских жалоб и петиций, а также для привлечения внимания жителей и администрации к актуальным проблемам города.

## 2. Роли пользователей

- `Гость` - просматривает жалобы и петиции.
- `Пользователь` - регистрируется, публикует жалобы, создает петиции, голосует.
- `Администратор` - модерирует публикации и управляет статусами.

## 3. Функциональные требования

### 3.1 Авторизация и регистрация

- пользователь может зарегистрироваться по имени, электронной почте и паролю;
- пользователь может войти в систему;
- система должна различать обычного пользователя и администратора.

### 3.2 Жалобы

- пользователь может создать жалобу;
- жалоба содержит заголовок, категорию, район, описание, дату и статус;
- жалобы отображаются в общем списке;
- администратор может менять статус жалобы.

### 3.3 Петиции

- пользователь может создать петицию;
- петиция содержит заголовок, описание, категорию, цель по голосам и автора;
- пользователи могут голосовать за петицию;
- один пользователь может проголосовать только один раз за одну петицию;
- администратор может управлять статусом петиции.

### 3.4 Панель администратора

- просмотр всех жалоб и петиций;
- изменение статусов публикаций;
- контроль за содержимым платформы.

## 4. Нефункциональные требования

- простой и понятный интерфейс;
- адаптивность для телефона и компьютера;
- защита пользовательских данных;
- устойчивость к некорректному вводу;
- расширяемая структура для дальнейшей серверной реализации.

## 5. Пользовательские сценарии

- житель регистрируется и отправляет жалобу о плохом освещении;
- пользователь создает петицию о ремонте дороги;
- другие жители поддерживают петицию голосами;
- администратор проверяет публикации и обновляет статусы.
