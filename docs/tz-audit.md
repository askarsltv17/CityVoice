# Сверка с CityVoice_TZ.docx

## Что соответствует ТЗ

- есть регистрация и вход пользователей;
- есть база данных пользователей, жалоб и петиций;
- есть создание, редактирование и удаление жалоб;
- есть создание, редактирование и удаление петиций;
- есть голосование за петиции с защитой от повторного голоса;
- есть общий список жалоб и петиций;
- есть админские действия по изменению статусов;
- есть модерация контента через репорты;
- интерфейс адаптивный для разных размеров экрана;
- проект работает на `Python + Flask + PostgreSQL`, что подходит под ТЗ.

## Что реализовано сверх ТЗ

- комментарии;
- реакции;
- уведомления;
- восстановление пароля по email;
- роли `moderator` и `admin`;
- карта с точками жалоб;
- демо-наполнение через `data/demo_seed.json`.

## Что не полностью совпадает с ТЗ

### 1. Блокировка пользователей

В ТЗ указана блокировка пользователей в админ-панели.

Сейчас реализовано:

- просмотр пользователей;
- смена роли пользователя.

Отдельного статуса блокировки или деактивации аккаунта пока нет.

### 2. Фильтрация по дате

В ТЗ упомянута фильтрация по категориям, дате и популярности.

Сейчас реализовано:

- поиск;
- фильтрация по категории;
- фильтрация по району;
- сортировка по новизне, голосам и популярности.

Отдельного UI-фильтра именно по дате пока нет.

## Итоговая оценка

Проект в целом соответствует основному ТЗ и даже заметно его расширяет.

Если оценивать по обязательному минимуму, реализация близка к полной.
Если оценивать строго по формулировкам ТЗ, остаются два заметных расхождения:

- нет отдельной блокировки пользователей;
- нет отдельного фильтра по дате.
